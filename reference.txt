Remainder-Update: 

Input:
When creating a reminder, users input:

Title - Name of the reminder (e.g., "Take medication", "Doctor appointment")
Description - Optional details about the reminder
Reminder Type - Category (e.g., "medication", "appointment", "task")
Reminder Date & Time - When they want to be reminded
Notification Timing - When to notify them:
"on-time" - Notify exactly at the scheduled time
"15-minutes-before" - Notify 15 minutes early
"1-hour-before" - Notify 1 hour early
"1-day-before" - Notify 1 day early
"custom" - Notify at a custom interval
Notification Channels - How to receive it:
In-app notification
Email
SMS (optional)
Priority - low/medium/high
Location (optional)
Recurring Pattern (optional) - daily, weekly, monthly, etc.


Output: 
When a reminder triggers, users receive:

Real-time In-App Notification - Socket.IO notification appears in their app immediately
Email Notification - Email sent to their registered email address (if enabled)
Notification Record - The system records:
When it was sent
Which channel it was sent through
Confirmation it reached the user
Example User Journey
User creates: "Take medication" reminder for Nov 19, 2:30 PM, notify 15 minutes before, via email + in-app

System delivers:

Nov 19, 2:15 PM → In-app popup + Email notification arrives
Reminder status changes from "pending" → "sent"
User can mark it as "completed" or "dismissed"
